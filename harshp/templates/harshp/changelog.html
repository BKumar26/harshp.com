v2.0 (2015-09-02)

  • Fixes #81 404/500 errors do not work in subdomains. [Harshvardhan Pandit]

    The subdomains have their own separate urls that do not get loaded if the subdomain is accessed directly, which is why the handlers for 404 and 500 are also not loaded. This is resolved by explicitly adding these handlers to the urls of every subdomain.

  • Fixes #78 date_end gets reset to date_start. [Harshvardhan Pandit]

    Hobby models had a bug in save() whereby the condition to check if the starting date was after the ending date was checked the other way round. This caused the ending date to be reset to the starting date everytime the model was saved. Added tests to ensure that the save method works as expected.

  • Fixes #82 feedback_add url does not work in prod. [Harshvardhan Pandit]

    The feedback_add url has an optional parameter for passing urls that can be blank. The misconfigured url regex required there to be two trailing slashes at the end, which caused the 404 error in prod. Fixed by omitting the trailing slash since the url itself contains a trailing slash when given.

  • Fixes #80 fabfile.deploy incorrectr req file. [Harshvardhan Pandit]

    Fabfile deploy() uses the correct production requirements file at requirements/requirements_prod.txt instead of the old value.

  • Django-filer as filebrowser, fabric sync scripts. [Harshvardhan Pandit]

    1. Uses django-filer as the filebrowser through a dummy model called FileUpload in sitedata. The addition of this model has been disabled since there is no pk field in the admin form. The model contains a FileField used for file uploads.
    2. Fabric now has scripts for pip-tools compile and sync. Tests are now run individually for each module, and combined using coverage.
    3. Added dev_noaws and dev_nodb files for using dev settings without either aws or db (ssh tunnel).
    4. Updated tests for general bug-fixes and optimisations.
    5. The file testruns.txt contains (upto) 100 test runs with maximum time taken for execution. This is compiled from the individual tests run by coverage.
